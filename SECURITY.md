# Security Policy / 安全策略

## Supported Versions / 支持版本

This project currently supports the latest commit on the default branch.  
当前项目仅支持默认分支上的最新提交版本。

## Reporting a Vulnerability / 漏洞报告方式

Please do not disclose security vulnerabilities publicly in issues.  
请不要在公开 Issue 中披露安全漏洞。

Send a private report with / 私下报告请包含：

- Affected component and version/commit / 受影响组件与版本（或提交号）
- Reproduction steps or proof of concept / 复现步骤或 PoC
- Potential impact assessment / 潜在影响评估
- Suggested remediation (if available) / 修复建议（如有）

Until a dedicated security mailbox is configured, open a private maintainer
contact request through the repository hosting platform.  
在专用安全邮箱配置完成前，请通过仓库托管平台向维护者发起私密联系请求。

## Disclosure Process / 披露处理流程

- We will acknowledge receipt as soon as practical.  
  我们会在可行范围内尽快确认收到报告。
- We will validate and triage the report.  
  我们会对报告进行验证与分级。
- We will coordinate a fix and responsible disclosure timeline.  
  我们会协调修复并制定负责任披露时间线。
